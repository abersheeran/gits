## 思考方式

- 以第一性原理质疑需求，防止我提出 X-Y 需求。
- 以全局视角思考解决方案，而不是着眼于局部。

## PRD

- 开始规划或修改前必须阅读相关的 PRD 文档
- 完成规划或修改后必须写入相关的 PRD 文档
  - 非功能性修改不需要写入 PRD 文档

## 代码规范

不要编写过长的代码文件，让人类和 AI Agent 都能更好的阅读、查找、修改代码。
遇到代码文件过长时，应主动拆解。

## 界面样式

- 样式必须使用 skills/design-system。
- 尽可能复用组件，如果必须自己编写组件则写入 web/src/components/common 文件夹中并添加相应的内容到 README.md 里指引后续的复用。
- 界面文案必须面向用户任务与结果，不允许写布局原因、实现细节、内部状态收敛、数据存储来源等自述性说明。
- 功能堆叠时不允许直接平铺所有元素在同一个界面上：
  - 可编辑的元素必须将当前态、编辑态放在同一空间并通过 icon button 或直接点击进行切换。
  - 长列表类元素必须有翻页或下拉加载设计，并考虑与其他元素的长、宽影响。
  - 创建或编辑表单、以及二次确认表单需要善用 Sheet、Dialog 等 shadcn 组件。
- 不允许出现指示性横幅。如果需要指引、解释功能，应当使用 ? 的 icon 并附加 tooltip。
- 不允许多次嵌套导致出现边框套边框、边距加边距的设计。
